# Chokepoint Radar — A股瓶颈投资信号仪表盘

> **MVP需求文档 v1.0** | 2026-06-05  
> 作者：熊国强 | 项目定位：AI PM转型作品集 + 个人实战工具

---

## 一、项目背景

### 1.1 我是谁
IT工程师转型AI产品经理中。每天用瓶颈投资法（Chokepoint Theory）做A股科技股分析，但流程全靠手动——翻公告、查数据、做表格、写判断，效率低且容易遗漏信号。

### 1.2 为什么要做这个
**三个身份，三个需求：**
- **作为投资者**：我需要一个仪表盘，一目了然看到持仓标的的瓶颈信号变化
- **作为AI PM候选人**：我需要一个web app作品，证明我能从0到1做产品
- **作为内容创作者**：公众号/小红书需要可视化素材，数据图表比文字更有说服力

### 1.3 为什么别人也会用
A股做科技股投资的散户很多，但大多数只看K线和消息面。瓶颈投资法提供了一个结构化框架，但门槛高——需要理解产业链、判断卡位、追踪信号。把这个框架产品化，降低使用门槛，有真实需求。

---

## 二、产品定位

**一句话描述：** A股科技股瓶颈投资信号仪表盘——输入股票代码，输出瓶颈评分和信号变化。

**不是什么：**
- 不是炒股软件（不提供买卖建议）
- 不是行情软件（不做K线图）
- 不是研报平台（不做深度分析报告）

**核心差异化：** 市面上的股票工具都是"看价格看消息"，这个工具是"看产业链卡位看信号变化"。

## 四、核心功能（MVP范围）

### 4.1 功能一：瓶颈评分雷达图

**用户故事：** 作为投资者，输入一个股票代码，我想看到它在瓶颈投资法四个维度上的评分，快速判断它是否符合筛选标准。

**四个评分维度（基于已有框架）：**

| 维度 | 评分依据 | 数据来源 |
|------|---------|---------|
| 物理必需 | 该环节是否下游必采？是否有替代方案？ | 行业白皮书+技术路线图（MVP阶段：预设打分） |
| 寡头垄断 | 全球CR3份额？是否有新进入者？ | 市场份额数据（MVP阶段：预设打分） |
| 产能刚性 | 扩产周期多长？当前产能利用率？ | 公司公告+行业研报（MVP阶段：预设打分） |
| 小市值 | 当前市值在50-500亿区间？机构覆盖度？ | 实时市值数据（API） |

**交互：**
- 输入框：输入股票代码或名称（支持模糊搜索）
- 输出：雷达图（四维评分，1-5分制）+ 总分 + 一句话总结
- 点击维度可展开看评分依据

**MVP简化：** 物理必需/寡头垄断/产能刚性三个维度用预设数据（人工标注），小市值用实时API计算。

### 4.2 功能二：信号变化时间线

**用户故事：** 作为投资者，我想看到某个标的最近30天的关键信号事件（认证、扩产、资金流入等），判断催化剂是否出现。

**信号类型（5类）：**

| 信号类型 | 图标 | 说明 |
|---------|------|------|
| 🟢 大客户认证 | 认证 | 进入大客户供应链或认证通过 |
| 🔴 产能紧张 | 产能 | 涨价、交期延长、产能利用率>90% |
| 💰 资金流入 | 资金 | 北向资金、主力资金持续流入 |
| ⚠️ 地缘事件 | 地缘 | 出口管制、制裁、贸易政策变化 |
| 📊 财报信号 | 财报 | 收入拐点、利润超预期 |

**交互：**
- 每个标的页面上方显示信号时间线（最近30天）
- 信号用不同颜色和图标区分
- 点击信号可看详情
- 无信号时显示"近30天无新信号，逻辑暂未变化"

**MVP简化：** 信号数据用预设JSON（从已有追踪记录中提取），不做实时爬取。

### 4.3 功能三：产业链全景图

**用户故事：** 作为投资者，我想看到某个产业链从终端需求到最上游的完整结构，并标注每个环节的瓶颈程度和对应A股标的。

**展示内容：**
- 树状结构：AI算力 → 光模块 → 激光器 → 衬底材料 → 高纯原材料
- 每个节点显示：环节名称、瓶颈程度（高/中/低）、CR3、A股标的
- 点击节点可跳转到对应标的的雷达图
- 当前支持3条产业链：CPO/硅光、先进封装、AI电力

**交互：**
- 顶部Tab切换产业链
- 树状图可缩放和拖拽
- 瓶颈程度用颜色深浅表示（深红=高瓶颈，浅灰=低瓶颈）

**MVP简化：** 用预设的3条产业链数据，树状图用D3.js或ECharts渲染。

### 4.4 功能四：自选股看板

**用户故事：** 作为投资者，我想在一个页面上看到我所有自选股的瓶颈评分和最新信号，不用一个个查。

**展示内容：**
- 卡片式布局，每张卡片：股票名称+代码、四维雷达图缩略、最新信号（如果有）、瓶颈总分
- 支持排序：按瓶颈总分、按最新信号时间、按市值
- 支持筛选：按产业链、按信号类型

**MVP简化：** 自选股列表用本地存储（localStorage），不搞用户系统。

## 五、页面结构

```

├── 添加/删除自选股

```

## 六、数据方案

### 6.1 MVP阶段（纯前端，无需后端）

| 数据类型 | 来源 | 方式 |
|---------|------|------|
| 股票基本信息 | 腾讯/新浪行情API | 前端直接调用（CORS允许） |
| 实时市值 | 同上 | 实时 |
| 瓶颈评分 | 人工标注 | 预设JSON文件 |
| 信号事件 | 从已有追踪记录提取 | 预设JSON文件 |
| 产业链结构 | 基于已有框架 | 预设JSON文件 |

### 6.2 后续迭代
- 信号数据：接入东方财富/同花顺公告API自动更新
- 评分数据：引入AI辅助评分（基于公告文本分析）
- 用户系统：支持多用户自选股云端同步

## 七、技术栈

| 层级 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js 15 (App Router) | 2026主流，SSR+SEO友好，面试官一看就懂 |
| 语言 | TypeScript | 类型安全，AI PM展示工程能力 |
| UI | shadcn/ui + Tailwind CSS | 高星组件库（10万+stars），可定制，简历加分 |
| 图表 | ECharts | 雷达图+树状图支持好，中文生态完善 |
| 数据可视化 | D3.js（备选） | 如果ECharts树状图不够灵活 |
| 部署 | Vercel | Next.js官方平台，一键部署，免费额度够用 |
| 包管理 | pnpm | 速度快，磁盘占用小 |

## 八、设计风格

- **整体调性**：专业、冷静、工具感，像一个投资研究终端而不是消费级app
- **主色调**：深色背景（#0a0a0a）+ 亮橙色点缀（#ff6b00），呼应小红书账号视觉但更沉稳
- **数据密度**：高，一屏信息量大，但不杂乱
- **字体**：中文用系统默认，数字用等宽字体（JetBrains Mono）
- **参考**：OpenBB Terminal（金融终端）、Bloomberg Terminal（专业感）

## 十一、GitHub策略（面向AI PM面试）

### 11.1 README写法
README不是技术文档，是**产品文档**。面试官30秒扫完README就要明白：
1. 我为什么做这个（个人痛点→通用需求）
2. 它解决了什么问题（瓶颈投资法的产品化）
3. 我做了哪些产品决策（为什么选这四个维度、为什么信号分五类）
4. 技术选型的trade-off（为什么Next.js而不是Vue、为什么纯前端）

### 11.2 关键决策记录
在 `/docs/decisions/` 目录下记录每个重要产品决策：
- 为什么瓶颈评分用1-5分而不是百分比？
- 为什么MVP阶段三个维度用预设数据？
- 为什么先做雷达图而不是先做信号追踪？
- 为什么选ECharts而不是D3？

这些决策记录比代码更能体现PM思维。

### 11.3 持续迭代展示
- 每个版本打tag，展示产品演进
- GitHub Issues里记录用户反馈（包括自己的使用反馈）
- 用GitHub Projects做看板，展示项目管理能力

## 十三、风险与应对

| 风险 | 应对 |
|------|------|
| 行情API跨域限制 | 用Next.js API Routes做服务端代理 |
| 预设数据不够丰富 | 先做3条产业链共15-20个标的，够用 |
| 瓶颈评分主观性强 | 展示评分依据，允许用户自行调整权重 |
| 开发周期超预期 | 严格按里程碑交付，每周末可演示 |
