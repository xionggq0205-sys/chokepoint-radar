# ADR-002: MVP采用纯前端+预设JSON，不搭后端

## 状态
已采纳

## 背景
Chokepoint Radar需要展示产业链数据、瓶颈评分和股价。是否需要后端API？

## 决策
MVP阶段纯前端，瓶颈评分和信号用预设JSON，股价用免费前端API（腾讯/新浪行情接口）。

## 理由
1. **降低开发复杂度**：MVP目标是验证产品逻辑和视觉方案，不是搭建完整数据管道
2. **GitHub友好**：clone下来`npm install && npm run dev`就能跑，无依赖外部服务
3. **瓶颈评分本就低频更新**：人工判断为主，一周更新一次足够，不需要实时API
4. **股价API免费可用**：腾讯/新浪行情接口支持前端CORS调用，无需代理

## 后果
- 股价API可能不稳定或有频率限制 → 加fallback机制和loading状态
- 预设JSON需要手动更新 → 后续版本加CLI脚本自动生成

## 替代方案
- Next.js API Routes + 数据库：MVP阶段过度工程 | 外部数据服务：增加成本和依赖
